# Project Scope Boundaries

Purpose: durable boundary file for current, blocked, future, and evidence-bound scope.

## Approved Now

- internal scaffold work
- fixture/local-only content
- Law & Business first
- project-control documentation
- web/mobile internal shells
- source-level and command-level QA

## Blocked Until Explicit Approval

- backend
- database
- Firebase
- Airtable runtime
- deployment
- auth/login/user accounts
- payments/subscriptions
- scoring
- readiness
- pass/fail
- saved progress
- analytics
- public content
- Question Bank migration
- C10 public content
- EAS
- APK
- AAB
- iOS build
- android/
- ios/
- app-store material
- marketing/public launch material

## Future Possible But Not Approved

Future implementation may include richer content systems, backend, progress, scoring, deployment, build/distribution, admin/editorial workflow, or monetization only after explicit approval and evidence gates.

## Never Claim Without Evidence

Do not claim readiness, completion, public status, production status, build status, install status, legal/currentness status, or content coverage unless direct evidence supports that exact claim.

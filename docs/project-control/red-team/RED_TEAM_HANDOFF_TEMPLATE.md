# Red-Team Handoff Template

Use this structure for red-team review responses.

## GitHub Truth Check

PR metadata and status.

## Changed Files Check

Changed file count and filenames.

## Source / Report Inspection

Relevant source and report evidence.

## Negative-Scope / Repo-Pollution Check

Forbidden files, unexpected paths, and dependency/lockfile status.

## Claim-Level Review

Claim level supported by current evidence.

## Risk Register Review

Active, new, or resolved risks.

## Decision

One approved review decision label.

## Next Allowed Step

Merge, patch, stop, main verification, or next phase only if authorized.

## Purpose And Authority Boundary — Documentation Scope

This is the canonical durable register for owner-accepted Red-Team findings. It records governance and traceability only. It does not authorize implementation, GitHub settings changes, connector permission changes, security-feature changes, workflow changes, control-script changes, product work, H0 closeout, H1, H7A, H8, approval, merge, production, or automatic continuation.

## Preservation, Supersession, And Resolution Contract

A finding must not disappear or be treated as resolved through silence. Supersession requires the exact superseding finding, rule, or decision identifier; exact owner authority; the explicit conflict or changed premise; preserved historical evidence; and revalidation of every affected dependency. Silence, chronology, a copied prompt, changed UI, PR creation, developer claims, or documentation edits alone do not supersede a finding.

A finding cannot become `COMPLETED_AND_VERIFIED` without every applicable item below:

- linked implementation issue;
- implementation pull request;
- exact reviewed head SHA;
- validation results;
- independent Red-Team result;
- required human approval;
- protected merge;
- current-main verification;
- applicable issue closeout;
- fresh live evidence for mutable GitHub state.

Deletion, ID reuse, and history rewriting are prohibited. Corrections add a new version or exact supersession edge while retaining the prior record.

### Conditional Valid-Deferral Extension

Issue #79 controls deferral requirements. This findings register records compliance and does not replace the master roadmap. Whenever `CURRENT_STATUS=VALIDLY_DEFERRED`, the record must also contain all seven nonempty fields below:

Empty, placeholder, circular, unresolvable, expired, or Issue-#79-incompatible values make the deferral invalid and block the responsible gate.

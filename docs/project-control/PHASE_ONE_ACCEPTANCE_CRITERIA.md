# Phase One Acceptance Criteria

Phase One acceptance does not equal production readiness, public readiness, app-store readiness, APK readiness, or legal/content currentness.

## Level 1 — Source/control completeness

- all internal-only warnings visible
- Law & Business internal fixture path present
- C10 deferred/blocked visible
- local fixture question interaction works
- feedback appears only after submit
- no score/readiness/pass-fail
- project-control logs exist
- risk register current
- claim levels honored
- known gaps documented

## Level 2 — Local command verification

- install/help/version commands run or limitations documented
- source behavior inspected
- forbidden-scope scan completed
- lockfile contamination scan completed when applicable

## Level 3 — Runtime visual verification

- internal warnings visually observed
- Law & Business track visually observed
- C10 deferred/blocked visually observed
- local fixture question selection visually observed
- feedback appears only after submit
- no outcome claim shown

## Level 4 — Internal build artifact, only when approved later

- approved build process completed
- build artifact identified
- unauthorized native or distribution scope excluded
- artifact indexed under approved artifact policy

## Boundary

Tests prove only what they directly test. Passing one level does not imply higher-level readiness.

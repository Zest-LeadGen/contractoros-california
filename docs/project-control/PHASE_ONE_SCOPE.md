# Phase One Scope

## Definition

Phase One is the internal ContractorOS testing-system MVP / scaffold.

Phase One is:

- Law & Business first
- fixture/local-only content
- internal-only
- not public
- not exam-ready
- no readiness/pass-fail claims
- no production/public MVP claim

## In Scope

- web internal shell already present
- mobile internal scaffold already present
- project-control reports
- ledger/risk/decision/artifact controls
- local fixture interaction
- internal warnings
- Law & Business fixture focus
- C10 deferred/blocked state

## Out of Scope

- backend
- database
- Firebase
- Airtable runtime
- deployment
- auth/login/user accounts
- payments/subscriptions
- scoring
- readiness
- pass/fail
- saved progress
- analytics
- public MCQs
- Question Bank migration
- C10 public content
- EAS/APK/AAB/iOS build
- android/ folder
- ios/ folder
- app-store materials
- public launch

## Claim Boundary

Phase One scope defines internal scaffold boundaries only. It does not establish production, public, legal/currentness, app-store, APK, or install readiness.

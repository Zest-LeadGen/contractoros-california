# Red-Team Recovery Playbook

Purpose: recovery process when project drift, unclear evidence, or overclaim risk appears.

## Steps

1. Stop.
2. Do not approve merge.
3. Downgrade claims.
4. Re-read DEVELOPMENT_CONTROL_MODEL_V3.md.
5. Re-read ROLE_BOUNDARIES.md.
6. Re-read PROJECT_SCOPE_BOUNDARIES.md.
7. Re-read PHASE_ONE_SCOPE.md if Phase One is involved.
8. Verify GitHub PR metadata.
9. Verify changed files.
10. Inspect changed files.
11. Check forbidden scope.
12. Rebuild the current state from evidence.
13. Issue one of the approved review decisions.

## Output Rule

State what evidence was found, what evidence is missing, and what the next allowed step is.
